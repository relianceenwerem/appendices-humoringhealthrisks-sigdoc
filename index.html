About Scribe

Scribe is built on generative AI (specifically large language models, using OpenAI's LLM APIs under the hood) combined with computer-vision-based process capture; a browser extension/desktop app that watches your screen activity (clicks, keystrokes, navigation) and automatically turns it into an annotated, step-by-step guide with screenshots.

This isn't a single proprietary algorithm unique to Scribe; it is a category of technology sometimes called "process AI" or "workflow capture AI." However, Scribe is unique because of the pairing of two AI functions:

  • Automated capture AI — turns literal screen actions into a structured visual guide (screenshots + auto-generated step text).
  • Generative "Scribe AI" — a separate LLM layer that writes surrounding context (titles, descriptions, tone edits, and full process documents assembled from multiple captured guides).

You can use Scribe capture to create documentation for various tasks, such as illustrating how to fill a form, use a program or software, apply for an opportunity, navigate a web page, etc.

Sharing & Embedding a Scribe

Beyond exporting a static file, Scribe offers several ways to share a document while keeping it live and interactive.

Link Sharing

Every Scribe (or Page) can be shared directly by link, with a permission level you control:

  • Not shared — viewable and editable only by you.
  • Team access — anyone on your Scribe team can find and view it; you can toggle view/edit permissions per document.
  • Organization access (Enterprise only) — any authenticated, licensed user in your organization can find and view it.
  • Anyone with the link — viewable by anyone who has the link, account or not.

You can also link directly to a specific step within a Scribe, so a viewer lands exactly where they need to be instead of scrolling through the whole guide.

Embed View Modes

When you embed a Scribe into a website, knowledge base, or intranet (as this page does), you can choose how it displays:

  • Scroll — the format used in this tutorial; steps flow continuously down the page.
  • Slides — one step at a time, with next/previous navigation, similar to a presentation.
  • Movie — an auto-playing, video-like walkthrough of the steps that updates automatically whenever the original Scribe is edited.

To set this, go to Share → Embed, choose Slides, Movie, or Scroll, then copy the embed code into your page.

Export Options in Scribe (ScribeHow)

For a standard Scribe (individual guide), you can export a static copy to:

  • PDF — a good option for offline or printable versions. Exported files are static, so any edits to the original Scribe require a new export.
  • Microsoft Word (DOCX) — useful when colleagues or clients need an editable offline version. Also static once exported.
  • HTML — code you can embed directly into a website, knowledge base, or intranet, preserving the rich formatting and interactivity of the original Scribe.
  • Markdown — a clean, lightweight version favored by developers, ideal for platforms like GitHub or Confluence since it strips away complex styling.

Notes

  • Plan requirement: PDF, DOCX, and Markdown export are available on Pro and Enterprise plans, not the free tier.
  • Enterprise admin lock: On Enterprise accounts, PDF and DOCX export are disabled by default and must be enabled by a System Admin.
  • Pages are different from Scribes: Pages (Scribe's multi-guide document format) can't be exported as PDFs, since they're built as live, interactive experiences that static files can't replicate. The standard PDF/HTML/Markdown export options are not currently available for Pages at all.
  • Export vs. share vs. embed: Exporting creates a static file (PDF, DOCX, Markdown, HTML snippet); sharing sends a live link with permission controls; embedding places the live Scribe directly on a page in Scroll, Slides, or Movie view. See the "Sharing & Embedding" section above for details on the latter two.

When to Use Each Option

Option Best for
PDF export Printing, sharing a fixed snapshot with external parties
DOCX export Handing off to someone who needs to edit the text in Word
HTML export Pasting formatted content into another editor (e.g. Google Docs)
Markdown export Pasting into Notion, Confluence, GitHub, or other dev-facing docs
Link sharing Sending a live, always-current guide to specific people or teams
Embed – Scroll Long-form documentation pages, like this tutorial
Embed – Slides Step-by-step walkthroughs where readers move at their own pace
Embed – Movie A hands-off, video-style walkthrough embedded on a page
